import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;
import org.springframework.stereotype.Service;

import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;
import java.util.List;

@Service
public class MyEntityService {

    @Autowired
    private MyEntityRepository myEntityRepository;

    public void exportDataToCsv(String fileName) {
        int pageSize = 1000; // 每次读取1000条记录
        int pageNumber = 0;

        try (BufferedWriter writer = new BufferedWriter(new FileWriter(fileName))) {
            // 写CSV头部
            writer.write("id,name,otherField");
            writer.newLine();

            while (true) {
                // 分页查询
                Page<MyEntity> page = myEntityRepository.findAll(PageRequest.of(pageNumber, pageSize));
                List<MyEntity> entities = page.getContent();

                // 如果没有更多数据，跳出循环
                if (entities.isEmpty()) {
                    break;
                }

                // 写入CSV
                for (MyEntity entity : entities) {
                    writer.write(entity.getId() + "," + entity.getName() + "," + entity.getOtherField());
                    writer.newLine();
                }

                // 下一页
                pageNumber++;
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }

ompt: Analyze DB2 Stored Procedure SQL Patterns in a Folder

You are a senior DB2 SQL PL / Stored Procedure expert and a static SQL lineage analysis expert.

## Goal

Analyze all DB2 SQL / Stored Procedure files under a given folder.

The purpose is **not** to list every file one by one.

The purpose is to identify what kinds of DB2 SQL PL patterns are used in the folder.

If multiple files use the same or very similar pattern, keep only **one most complex and most representative example**.

The final output should be a Markdown report in English.

---

## Input

Folder path:

```text
<PUT_FOLDER_PATH_HERE>
```

Example:

```text
D:\check\sample_case\sp
```

File extensions may include:

```text
.sql
.db2
.ddl
```

Please scan the folder recursively.

---

## Main Task

For each SQL file:

1. Detect whether it contains DB2 Stored Procedure / Function / SQL PL code.
2. Identify the DB2 SQL PL syntax patterns used in the file.
3. Group files by syntax / usage pattern.
4. Deduplicate similar patterns.
5. For each pattern, keep only the most complex and most useful representative example.
6. Explain why that file is selected as the representative example.
7. Extract the most important SQL snippet from the file.

## Deduplication Rules

If several files contain the same pattern, do **not** output all of them.

Select only the best representative file using these rules:

1. Prefer the file with the highest syntax complexity.
2. Prefer the file that combines multiple related patterns.
3. Prefer the file whose SQL snippet is complete and useful as a test case.
4. Prefer real business-like logic over trivial demo SQL.
5. If two files are similar, choose the one with:
   - more lines,
   - deeper nesting,
   - more DML types,
   - more joins,
   - more predicate logic,
   - more lineage value.
